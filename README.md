# 声历 Voice Calendar

语音优先的日历管理工具，面向经常在移动、开会、做家务或双手被占用时管理日程的用户。核心目标是让用户用一句中文自然语言完成添加、删除、查看和提醒管理。

## 真实需求拆解

- 快速录入：用户不想逐项点选日期、时间、提醒方式，希望直接说“明天下午三点开会”。
- 容错理解：中文语音里常见“下周五”“半小时后”“十点半”“提前二十分钟”等说法，工具需要能稳定解析。
- 低打扰提醒：提醒应能在页面内、语音播报和系统通知之间平滑工作。
- 可复查可纠错：识别文本必须可编辑，语音没听准时可以手动改后再执行。
- 离线可演示：评审环境不一定能联网，基础功能应不依赖后端服务。

## 已实现功能

- 语音识别：使用浏览器 Web Speech API，支持中文语音转文字。
- 语音反馈：使用 SpeechSynthesis 播报添加、删除、查询和提醒结果。
- 中文命令解析：支持相对日期、周几、显式日期、上午/下午、半小时后、提前提醒。
- 多轮追问：添加日程时缺少日期、时间或事项名称，会继续追问补齐后再创建。
- 日历操作：支持添加、删除、查看今天/明天/本周/本月/全部日程。
- 月历查看：支持选择不同年份、月份，上一月/下一月切换，并快速回到今天。
- 节假日标注：已录入 2026 年中国法定节假日和调休上班日，在月历中显示“休/班”标记。
- 本地提醒：页面内 Toast、语音播报，可选浏览器系统通知。
- 主题切换：支持浅色、深色、春、夏、秋、冬六种主题，并保存用户偏好。
- 本地存储：事件保存在 LocalStorage，刷新页面后仍可查看。
- 可测试核心：自然语言解析和事件匹配逻辑位于 `src/calendar-core.js`，配有 Node 测试。

## 依赖说明

本项目没有引入第三方运行时库或框架，依赖能力均来自浏览器和 Node.js：

- Web Speech API：浏览器语音识别，部分浏览器可能不支持。
- MediaDevices.getUserMedia：启动语音前用于请求和校验麦克风权限。
- SpeechSynthesis：浏览器语音播报。
- Notification API：系统通知，可由用户授权开启。
- LocalStorage：本地事件存储。
- Node.js：仅用于本地静态服务器和测试脚本。

语音识别需要 HTTPS 或 localhost 环境，并依赖浏览器自身的语音服务。若浏览器提示权限、麦克风、网络或未检测到语音问题，可先使用文字命令完成同样操作。

## 节假日数据

当前节假日数据来自中国政府网发布的《国务院办公厅关于2026年部分节假日安排的通知》，本地维护在 `src/holiday-data.js`。后续年份需要在官方通知发布后继续补充。

原创功能部分：

- 中文自然语言命令解析。
- 日程增删查、范围筛选和模糊删除匹配。
- 本地提醒调度、语音反馈和页面交互。
- 前端界面、样式和测试用例。

1. `feat: 初始化语音日历静态应用`

3. `feat: 接入语音识别和语音反馈`
4. `feat: 增加本地提醒和通知授权`
5. `test: 覆盖日程解析与删除匹配`
6. `feat: 增加主题切换功能`

每个 PR 都应包含功能描述、实现思路和测试方式。仓库已提供 `.github/pull_request_template.md` 作为提交模板。
